# BigQuery Data Analysis & Charting Agent

A data analysis and charting agent built using the Google Agent Development Kit (`google-adk`). It accesses BigQuery tables, executes Python code using the built-in Gemini code interpreter to manipulate data and generate charts, and automatically captures the plotted charts as session artifacts to render them in the ADK Web UI.

It supports:

1. **Dynamic Setup & Configuration**: Automatically prompts the user for configuration details and saves them in the session state.
2. **Built-in Python Sandbox**: Executes generated Python code inside the Gemini code executor to perform data manipulations and plot charts.
3. **Automatic Artifact Capture**: Captures generated figures (e.g. `chart.png`) directly from the code execution context and shows them in the web artifact viewer.

---

## Architecture

- **`agent.py`**: Configures a **`root_agent`** (`LlmAgent`) that acts as a coordinator, delegating tasks to two specialized sub-agents:
  1. **`data_engineer_agent`**: Handles database configuration and querying via:
     - `BigQueryToolset`: Native ADK BigQuery toolset to inspect schemas, list tables, and execute SQL queries.
     - `save_configuration`: Saves project metadata into session state.
     - `save_data_for_visualizer`: Strictly saves extracted data to hand off to the visualizer.
  2. **`visualizer`**: Dedicated plotting agent running:
     - `BuiltInCodeExecutor`: ADK code executor running Python code generated by the LLM in the Gemini sandbox to plot and save `chart.png`.

---

## Prerequisites

- Python 3.12
- Google Cloud Project credentials configured (Application Default Credentials - ADC) in the environment.

---

## Running the Agent via ADK Web

To run the agent in the ADK Web UI:`

```bash
# Start the ADK web server targeting the project directory
uv run adk web data_analysis_agent
```

Once the Web UI is open in your browser, you can chat with the agent.

---

## Example Interaction Flow (Dynamic Configuration & Charting)

### 1. Initial Setup and Configuration

When you start a new conversation, the agent will prompt you to set up your connection parameters:

- **You**: `I want to get set up.`
- **Agent**: `Sure! I can help you configure your BigQuery data analysis environment. Please provide me with the following details: 1) GCP Project ID, 2) BigQuery Dataset ID, and 3) Dataset Location.`
- **You**: `Use project: my-gcp-project, dataset: sales_data, location: US.`
- **Agent**: (Calls `save_configuration(...)` to save the values to the session state). `Configuration saved successfully! We are ready to start querying and plotting.`

### 2. Querying & Charting (Multi-Turn Example)

- **You**: `Show me the schema of the monthly_revenue table and plot a line chart of the revenue over the last 12 months.`
- **Agent**:
  - Automatically queries the schema (e.g., calling BigQuery tools with the saved `project_id` and `dataset_id`).
  - Executes a SQL query to fetch the last 12 rows.
  - Generates Python code to plot the data using matplotlib and saves it to `chart.png` inside the sandbox.
  - The `BuiltInCodeExecutor` captures the output `chart.png` and displays the image directly in the browser's artifact panel.

- **You**: `Great! Now add a trend line to that chart and make the main line red.`
- **Agent**:
  - Automatically updates the Python code to compute a trend line and render the primary line in red, saving the figure back to `chart.png` to refresh the artifact panel.
